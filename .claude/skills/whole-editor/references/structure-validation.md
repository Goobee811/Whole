# Structure Validation Checklist

## Pre-Edit Validation
- [ ] Target section identified clearly
- [ ] Domain context understood
- [ ] Function context understood
- [ ] Existing cross-refs mapped

## Content Validation

### 4-Point Structure
- [ ] Definition present and clear
- [ ] Context explains placement
- [ ] Application provides examples
- [ ] Integration links to other concepts

### Bilingual Format
- [ ] Vietnamese primary text present
- [ ] English translation accurate
- [ ] Format: `## English | Tiếng Việt`

### Cross-References
- [ ] All refs use: `Domain > Function > Concept`
- [ ] Bidirectional refs established
- [ ] No orphaned refs

## Post-Edit Validation
- [ ] All existing content preserved or approved for change
- [ ] New content follows 4-point structure
- [ ] Bilingual format maintained
- [ ] Cross-refs updated bidirectionally
- [ ] No broken links
- [ ] Changes align with "only add, never subtract"
